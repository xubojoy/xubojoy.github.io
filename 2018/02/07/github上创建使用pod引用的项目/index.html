github上创建使用pod引用的项目

这两天折腾cocoapods，折腾的也是够够的🐶，各种尝试发布自己的框架到github上，供其他人直接用pod安装。现在记录下过程：2018-11-13有更新

首先必备技能：玩转git及github

一、打开www.github.com,创建自己的项目仓库

大致就是这个样子，本文仓库名JOYGCDTimer
有关开原协议，这里采用Apache License, Version 2.0,然后使用git clone [仓库地址],将仓库克隆到本地。
二、将要测试的demo及要发布的开元框架放到仓库JOYGCDTimer里,并创建.podspec文件

.podspec文件的创建：

pod spec create JOYGCDTimer


此时的FJAlertView就是之后可以被搜索的pod名字

三、配置.podspec文件，这是我使用的是Sublime Text

关于s.source_files设置可参考

  • name:类库的名称
  • version:库的版本
  • summary: 库的介绍
  • homtepage:Github上项目地址
  • license:许可证
  • author:作者
  • platform:运行的平台及最低支持版本
  • source:Github上项目的https链接地址
  • source_files:要共享的代码，这里是JOYGCDTimer下面的所有代码。
  • dependency：第三方依赖库

四、创建相应的分支及tag标签

五、在收到的邮件里面有一个 Getting setup with Trunk，注册 trunk,它的作用是能够让我们把本地的 spec 文件上传到 gitHub的远程索引库

pod trunk register xubojoy1112@163.com 'xubojoy' --description="倒计时" --verbose

打开邮件连接，之后返回到终端执行以下命令

pod trunk push JOYGCDTimer.podspec


此时已基本成功了！

六、验证podspec

pod spec lint JOYGCDTimer.podspec

七、执行验证命令：

pod lib lint JOYGCDTimer.podspec

如上则就是验证成功！

八、pod search可能遇到的错误

进入~/资源库/Caches/CocoaPods/search_index.json找到search_index.json文件并删除，之后重新pod search JOYGCDTimer

整个过程到此完结，可以开始装逼了！！！🙃🙃🙃🙃🙃———-
